Explain the Android Software Stack / Architecture layers
5 marks3×
▼
202220232025
Android architecture has 4 main layers from bottom to top:

Layer 1 — Linux Kernel: Foundation. Handles hardware abstraction, security, memory management, process management, drivers (Display, Camera, WiFi, Audio)
Layer 2 — Native Libraries + Android Runtime: C/C++ libraries: WebKit, SQLite, FreeType, OpenGL ES, Media Framework, SSL. DVM/ART runs application code
Layer 3 — Application Framework (Java API): Activity Manager, Window Manager, Content Providers, View System, Notification Manager, Resource Manager. Developers interact with this layer
Layer 4 — Applications (Top): Native and third-party apps (Phone, Contacts, Browser, Email, Camera)
Unit I — Essay (10–11 marks)
Explain the Android Application Lifecycle in detail
10–11 marks5×
▼
20212022202320242025
The Activity Lifecycle determines the state of an Activity from creation to destruction. Android invokes callback methods as the state changes.

Four States:
Active/Running — Foreground, has user focus
Paused — Another activity in foreground but this is still visible; retains state
Stopped — Completely obscured; no longer visible; can be killed by system
Destroyed — Removed from memory

Lifecycle Callback Methods:
MethodWhen CalledUse For
onCreate()Activity first createdInitialize UI, setContentView(), bind data
onStart()Becoming visible to userPrepare UI for display
onResume()Starts interacting with userStart animations, sensors, camera
onPause()About to resume another activitySave unsaved data, pause animations
onStop()Activity no longer visibleHeavy resource shutdown
onRestart()After stop, starting againRefresh data
onDestroy()Before activity is destroyedFinal cleanup
Lifecycle Flow:
Start → onCreate() → onStart() → onResume() → RUNNING User leaves → onPause() → onStop() → STOPPED User returns → onRestart() → onStart() → onResume() App closed → onStop() → onDestroy() → DESTROYED
onSaveInstanceState(Bundle) — Called before the activity is stopped to save temporary UI state (e.g., text in EditText) across configuration changes like screen rotation. The saved Bundle is passed back to onCreate() on restart.

Explain Intents in Android in detail with example
5 marks5×
▼
20212022202320242025
An Intent is an abstract description of an operation to be performed.

Intent Structure (6 components):
Component Name — specifies exact class (makes it Explicit)
Action — e.g., ACTION_VIEW, ACTION_SEND, ACTION_DIAL
Data — URI of data to operate on (e.g., tel:123, http://...)
Category — additional info (e.g., CATEGORY_LAUNCHER)
Extras — Key-value pairs (Bundle) for additional data
Flags — Metadata on how to launch the activity

Explicit Intent (within own app):
Intent i = new Intent(this, SecondActivity.class); i.putExtra("username", "Noor"); startActivity(i);
Implicit Intent (any capable app):
Intent i = new Intent(Intent.ACTION_VIEW, Uri.parse("http://www.google.com")); startActivity(i);
Intent Resolution for implicit intents: Android compares the intent with Intent Filters in other apps' manifests, testing Action, Category, and Data type.

Explain the different types of Layouts in Android in detail
10 marks4×
▼
2022202320242025
A Layout Manager defines the visual structure for a UI and handles positioning and sizing of child views.

1. LinearLayout
Arranges children in a single direction — vertical or horizontal
Key attribute: android:orientation="vertical" or "horizontal"
android:layout_weight distributes remaining space proportionally
<LinearLayout android:orientation="vertical"> <TextView android:text="Name" /> <EditText android:hint="Enter name" /> </LinearLayout>
2. RelativeLayout
Children positioned relative to each other or to the parent
Eliminates nested LinearLayouts → flatter, more efficient hierarchy
Key: layout_below, layout_toRightOf, alignParentTop, centerInParent
3. TableLayout
Arranges children in rows and columns like an HTML table
Uses <TableRow> elements; each view inside is a cell
stretchColumns, shrinkColumns, collapseColumns control column behavior
4. FrameLayout
Designed to hold a single item on screen
Multiple children stack (overlap) — most recently added is on top
Use cases: overlays, loading spinners, Fragment containers
5. GridLayout
Places children in a rectangular grid with arbitrary rows and columns
Children can span multiple rows/columns (rowSpan, columnSpan)
Flatter hierarchy than TableLayout (no TableRow needed)

Explain Database Manipulation using SQLite in Android
10 marks4×
▼
2021202220232024
SQLite is Android's built-in relational database. The android.database.sqlite package provides the API.

Key Classes:
SQLiteOpenHelper — Manages DB creation and version upgrades
SQLiteDatabase — Performs insert, update, delete, query
Cursor — Iterates through result rows row by row
ContentValues — Map of column name → value for insert/update

Step 1: Create the Helper Class
public class DBHelper extends SQLiteOpenHelper { static final String DB_NAME = "College.db"; static final int DB_VERSION = 1; static final String TABLE = "students"; public DBHelper(Context c) { super(c, DB_NAME, null, DB_VERSION); } @Override public void onCreate(SQLiteDatabase db) { db.execSQL("CREATE TABLE " + TABLE + " (" + "ID INTEGER PRIMARY KEY AUTOINCREMENT, " + "NAME TEXT, MARKS INTEGER)"); } @Override public void onUpgrade(SQLiteDatabase db, int o, int n) { db.execSQL("DROP TABLE IF EXISTS " + TABLE); onCreate(db); } }
Step 2: INSERT data
public boolean insertData(String name, int marks) { SQLiteDatabase db = this.getWritableDatabase(); ContentValues cv = new ContentValues(); cv.put("NAME", name); cv.put("MARKS", marks); long result = db.insert(TABLE, null, cv); return result != -1; // -1 = error }
Step 3: READ (SELECT) data
public Cursor getAllData() { SQLiteDatabase db = this.getReadableDatabase(); return db.rawQuery("SELECT * FROM " + TABLE, null); }
Step 4: UPDATE data
public boolean updateData(String id, String name, int marks) { SQLiteDatabase db = this.getWritableDatabase(); ContentValues cv = new ContentValues(); cv.put("NAME", name); cv.put("MARKS", marks); db.update(TABLE, cv, "ID=?", new String[]{id}); return true; }
Step 5: DELETE data
public int deleteData(String id) { SQLiteDatabase db = this.getWritableDatabase(); return db.delete(TABLE, "ID=?", new String[]{id}); }
SQLite Data Types:
TypeDescriptionJava Equivalent
INTEGERSigned integer (1–8 bytes)int, long
TEXTText string (UTF-8)String
REALFloating point numberfloat, double
